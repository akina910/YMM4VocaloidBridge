# Contributing

YMM4VocaloidBridgeへのIssue、設計提案、ドキュメント修正、コードのPull Requestを歓迎します。

## 開発前のルール

1. 大きな変更は、先にIssueまたはRFCを作成してください。
2. 対応するPhaseと受け入れ条件を明記してください。
3. 初音ミク、VOCALOID、YMM4などのバイナリ・画像・認証情報をコミットしないでください。
4. ライセンス回避、メモリ改変、暗号化された独自形式の解析を目的とする変更は受け付けません。
5. 上流コードを利用する場合は、リポジトリURL、コミットSHA、ライセンス、変更点をPRへ記載してください。

## ブランチとPull Request

- ブランチ名: `type/short-description`
- 例: `docs/midi-import-findings`、`feat/voice-detection`、`fix/export-timeout`
- 1つのPull Requestは1つの目的に絞ってください。
- Draft PRを早めに出して、設計段階から相談して構いません。
- メンテナーが編集できる設定を有効にしてください。

## PRに必要な情報

- 解決する問題と関連Issue
- 変更した範囲と変更していない範囲
- 自動テスト結果
- 手動確認環境と再現手順
- UI Automation変更時は、対象アプリの版・UI言語・DPI倍率
- 上流コードまたは素材を使った場合のライセンス情報

## Definition of Done

- 受け入れ条件が満たされている
- 追加・変更したロジックに相応のテストがある
- 失敗時に無期限待機せず、タイムアウト・キャンセル・診断情報を返す
- ボイスバンク、キャラクター素材、個人パス、認証情報を含まない
- 利用者向けの挙動変更がREADMEまたはdocsへ反映されている
- CIとレビューが通っている

## ライセンス

Pull Requestを提出することで、提出内容を本リポジトリのMIT Licenseの下で提供する権利を持ち、その条件で配布されることに同意したものとみなします。
