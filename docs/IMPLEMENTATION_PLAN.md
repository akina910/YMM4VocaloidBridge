# Implementation Plan

Status: **beta.4 quality acceptance in progress**

Last updated: 2026-07-14

この文書は実装と受入の基準計画です。beta.3では実YMM4の音声アイテム、ORIGINAL音声、YMM4ネイティブ口パクフレーム、キャッシュ再利用まで確認しましたが、生成音声の聞き取り品質を受入条件に含めていなかったため完了判定を撤回しました。beta.4は機械的E2Eに加えて複数文のASR比較とユーザーの実リスニングをリリースゲートにします。証跡は[M0 Feasibility Evidence](M0_FEASIBILITY_EVIDENCE.md)と[End-to-End Evidence](E2E_EVIDENCE.md)に記録します。ユーザー所有の立ち絵素材そのものは配布・複製しません。

## 1. Product Goal

YMM4の通常のセリフ入力から、ユーザーがローカルに所有する初音ミク V6を使った会話調音声を生成し、ユーザー所有の動く立ち絵へ口パクタイミングを適用できるようにします。

## 2. MVP

### 対象

- Windows 11
- YMM4 v4.47.0.0以降の現行安定版
- .NET 10
- VOCALOID6 Editor 6.12.0以降
- 初音ミク V6、日本語UI、日本語の短いセリフ
- ユーザーが用意した「あ・い・う・え・お・閉じ」の口画像

### 必須の利用体験

1. YMM4でセリフを入力する
2. 会話用の読み・モーラ・ノート列が決定的に生成される
3. VOCALOID6で所有済みボイスバンクを選び、WAVを生成する
4. WAVがYMM4の指定先へ返る
5. 同じセリフから口パクタイミングが生成される
6. 再生成時にキャッシュが利用される
7. 失敗時に理由と復旧手順が表示される

### MVPでは行わないこと

- ボイスバンク、エディタ、キャラクター画像の同梱
- VOCALOID6の代替となる歌唱エディタの実装
- VPRの独自解析や書き換え
- ライセンス認証、DRM、製品制限の回避
- macOS対応、多言語対応、リアルタイム合成
- 任意のVOCALOID6バージョン・UI言語への対応保証

## 3. Delivery Phases

### Phase 0: Feasibility and Boundaries

目的は、実装方式が成立することをコード着手前に確認することです。

- 現行YMM4で.NET 10の公式音声プラグインサンプルが読み込まれることを確認する
- `IVoiceSpeaker.CreateVoiceAsync`から外部処理を待機してもUIがデッドロックしないことを確認する
- 歌詞イベント付き標準MIDIをVOCALOID6 6.12.0へ読み込めることを確認する
- 初音ミク V6の選択、MIDIインポート、WAVエクスポートの操作経路を記録する
- UI Automationで取得できるコントロールと、座標操作が必要な領域を分類する
- 出力先を決定的に指定し、完了・失敗を検知できることを確認する
- YMM4内で口パク項目を追加できる公開APIの有無を確認する
- YMM4内で完結できない場合の`.lab`サイドカー方式を検証する
- 上流コード、製品EULA、商標・素材の配布境界を文書化する

**Exit gate**

- 各項目にIssue、再現手順、スクリーンショットまたはログがある
- 完全自動モードまたは手動補助モードの採用判断が記録されている
- 口パク統合方式が1つ選ばれている
- 禁止事項に触れず実装できると判断されている

成立しない項目がある場合は、機能を削るのではなく、手動補助モードなどの代替案をRFCで合意します。

### Phase 1: Walking Skeleton

- .NET 10ソリューションとテストプロジェクト
- YMM4で読み込める最小の`IVoicePlugin` / `IVoiceSpeaker`
- VOCALOID6と初音ミク V6のインストール検出
- 設定画面、診断コマンド、キャンセル、タイムアウト
- Windows CIでビルド・テスト・ライセンス検査

**Exit gate:** プラグインがYMM4に表示され、未設定時もクラッシュせず診断結果を返す。

### Phase 2: Assisted Audio Path

- 日本語テキストの正規化、読み、モーラ分割
- 会話向けの長さ・休符・ピッチ規則
- 歌詞付き標準MIDIの生成
- VOCALOID6を開き、ユーザー操作でWAVを書き出す補助フロー
- 出力WAVの監視、検証、YMM4への返却
- 入力・設定・製品版を含むキャッシュキー

**Exit gate:** 1つのセリフが手動補助つきでYMM4の音声アイテムになる。

### Phase 3: VOCALOID6 Automation

- プロセス起動・既存プロセス検出
- UI AutomationによるMIDI読込、ボイス選択、WAV書出し
- DPI、ウィンドウ状態、ダイアログ、処理中表示への耐性
- 操作ごとのタイムアウト、リトライ、キャンセル、診断ログ
- 自動操作失敗時の手動補助モードへの復帰

**Exit gate:** 対応環境のテストマトリクスで連続20セリフを処理し、ハングせず成功率95%以上。

### Phase 4: Lip Sync

- モーラと母音から口形列を生成する純粋ロジック
- シーケンス時刻から`.lab`互換タイミングを生成する
- 必要に応じてWAVの無音区間で開始・終了時刻を補正する
- Phase 0で選んだYMM4統合方式を実装する
- 「あ・い・う・え・お・閉じ」の割り当てUI
- 音声より1から2フレーム先行させる調整値

**Exit gate:** 短い日本語セリフ10件で、音声と立ち絵の口形が決定的に再現される。

### Phase 5: Packaging and Public Beta

- `.ymme`パッケージ生成
- 初回セットアップと診断画面
- Windows 11、日本語UI、複数DPIでのE2E検証
- 失敗ログの個人情報マスキング
- インストール、更新、アンインストール文書
- 既知の制限と上流ライセンス一覧

**Exit gate:** クリーン環境でインストールから1+3のワークフローを完走できる。

## 4. Test Strategy

### Unit tests

- テキスト正規化、モーラ分割、句読点と休符
- 会話プロソディの決定性
- MIDIイベントとタイムライン
- 口形変換と`.lab`出力
- キャッシュキー、パス検証、ログのマスキング

### Contract tests

- YMM4プラグインAPI境界
- VOCALOIDドライバーの状態遷移
- UI Automationアダプターを偽装した成功・失敗・タイムアウト

### Manual E2E matrix

- YMM4のサポート最小版と最新安定版
- VOCALOID6 6.12.0と確認時の最新安定版
- Windows表示倍率100%、125%、150%
- VOCALOID6起動済み・未起動
- 保存先が通常・空白入り・日本語・書込不可
- 1件、連続20件、キャンセル、再試行、キャッシュ再利用

## 5. Release Criteria

- PhaseごとのExit gateを満たす
- CIが成功する
- 未解決のCritical/High脆弱性がない
- ライセンスと第三者通知が更新されている
- 正規所有ソフトだけで再現できる手順がある
- プロプライエタリなバイナリ・素材・認証情報を含まない
- 少なくとも1件の実機E2E証跡がリリース候補に紐づいている
- 難読文を含む複数の実機WAVで発音内容を照合し、ユーザーが実際に試聴して受け入れている

## 6. Open Decisions

Phase 0で次を決定します。

- 完全自動と手動補助のどちらを初期既定にするか
- 日本語の読み変換ライブラリと辞書の配布方式
- MIDI歌詞イベントの文字コード
- 初期の会話ピッチ規則
- YMM4口パク統合をプラグイン内で行うか、サイドカー処理にするか
- UI Automationライブラリとバージョン
- サポートするYMM4最小バージョン
